# 슈퍼로봇대전 J — 전체 챕터 한글 대사 삽입 도구

GBA 『슈퍼로봇대전 J』(한글패치 ROM)의 **시나리오 대사 70개 챕터**를
한국어로 바꿔 넣는 PC용 파이썬 도구입니다.

번역 매칭 엑셀이 있는 챕터를 한 번에 모두 패치하며, 새 한글 대사가
들어갈 공간이 모자라면 **ROM 을 자동으로 확장**합니다.

---

## 1. 빠른 시작

### 준비물 (같은 폴더에)

| 파일 | 설명 |
|------|------|
| `*.gba` | 한글패치된 슈로대J ROM (입력, 1개) |
| `translations/` 폴더 | 번역 매칭 엑셀들을 모아둔 폴더 |
| `korea2350.txt` `japan2350.txt` | 한글↔한자 폰트 매핑 (동봉) |

번역 엑셀 이름은 **`srwj_matched_block_NNN.xlsx`** 형식입니다.
`NNN` 은 **archive 인덱스**입니다 (매칭 도구가 붙이는 번호 그대로).
대사 블록의 archive 인덱스는 `1, 3, 5, …, 137, 191` 이며,
`block_001` 이 1화입니다. 번역을 끝낸 챕터의 엑셀만 폴더에 두면
됩니다 (나머지는 일본어 유지). 파일 이름은 바꾸지 말고
매칭 도구가 만든 그대로 두세요.

### 실행

```
pip install openpyxl

# 일부 챕터만 (희생슬롯 방식, 안전)
python patch_all.py --xlsx-dir translations

# 전체/대부분 챕터 (사전 확장 방식)
python patch_all.py --xlsx-dir translations --expand-dict
```

결과물 `srwj_korean_all.gba` 가 생성됩니다.

### 옵션

* `--rom <파일>` 입력 ROM 지정 (기본: 폴더의 `*.gba`)
* `--out <파일>` 출력 ROM 이름
* `--reserve N` 첫 줄 `화자「` 가 차지하는 폭 (기본 7)
* `--addr 0x...` 새 블록 기록 시작 주소 (기본 `0xFB2400`)
* `--expand-dict` 사전을 물리적으로 확장 (아래 4-2 참고)
* `--keep-jp` 번역 없는 턴을 원본 일본어로 둠 (아래 3-2 참고)
* `--placeholder-text "..."` 미번역 턴 표시 문구 변경 (기본 `번역 없음`)

## 2. 동작 원리 (요약)

### 한글 = 한자 자리 바꿔치기
한글패치 ROM 의 폰트는 일본 한자 글리프(`亜`~`美`, 2350자) 자리에
한글 글자(`가`~`힝`)가 덮여 있습니다. 대사에 한자를 넣으면 화면엔
그 자리의 한글이 나옵니다.

```
한글 → 대응 한자 → 그 한자를 디코딩하는 사전 코드 → 게임 데이터
```

`korea2350.txt` 와 `japan2350.txt` 가 같은 줄 번호끼리 대응됩니다.

### ROM 삽입
한글 대사는 일본어보다 커서 원래 자리에 안 들어갑니다.
챕터별로 새 블록을 ROM 빈 공간에 차례로 기록하고, 아카이브 인덱스가
새 위치를 가리키도록 수정합니다. 블록 내부 구조(포인터 테이블+대사)는
그대로 재구성하므로 게임이 정상적으로 읽습니다.

---

## 3. 공간 부족 대책 — ROM 자동 확장

> **질문: 빈 공간이 모자라면?**

게임 전체 한글 대사는 약 **1.9 MB** 로 추정되는데, 원본 ROM 끝의
빈 공간은 **311 KB** 뿐입니다. 그대로는 다 못 넣습니다.

**대책: ROM 파일 자체를 확장합니다.**

* GBA 카트리지 주소공간은 **최대 32 MB** 까지 지원합니다.
  (현재 ROM 은 16 MB → 32 MB 로 키울 여지가 16 MB 있음)
* `patch_all.py` 는 새 블록을 빈 공간부터 쌓다가 16 MB 를 넘어가면
  ROM 파일을 **자동으로 32 MB 까지 확장**합니다 (2의 거듭제곱 단위).
* 확장된 ROM 은 에뮬레이터(mGBA, VBA 등)와 대부분의 플래시카트에서
  정상 동작합니다.

전체 70챕터를 넣어도 약 18 MB 면 충분하므로 32 MB 안에서 여유롭습니다.

> **다른 방식은?**
> 번역된 챕터의 '원래 자리'도 빈 공간이 되지만, 손대지 않는 이벤트
> 블록들과 잘게 뒤섞여 있어 재활용이 비효율적입니다. 게다가 그것까지
> 합쳐도(약 1.2 MB) 한글 전체(1.9 MB)에는 모자랍니다.
> → **ROM 확장이 가장 단순하고 확실한 답**입니다.

### 3-2. 번역이 없는 턴 — "번역 없음" 표시

번역 매칭이 안 됐거나 번역문이 비어 있는 턴은 **기본적으로 화면에
`번역 없음` 이라고 표시**됩니다. 게임을 진행하며 어느 턴이 아직
번역되지 않았는지 한눈에 알 수 있습니다.

* 표시 문구는 `--placeholder-text "..."` 로 바꿀 수 있습니다.
* `--keep-jp` 를 주면 미번역 턴을 원본 일본어 그대로 둡니다
  (일본어를 읽으며 검수하고 싶을 때).
* 패치 로그의 `미번역 표시 턴` 수와 챕터별 `⚠ 미번역 N` 으로
  몇 개가 남았는지 확인할 수 있습니다.

## 4. 사전(코덱) 용량과 그 대책

게임 사전은 한자 2350자 중 1398자만 직접 인코딩할 수 있습니다.
나머지는 사전 테이블을 덮어써서 해결하는데, 여기에 두 가지 방식이
있습니다.

### 4-1. 희생슬롯 방식 (기본)

`seg1`(코드 `0xC300`~) 의 '거의 안 쓰이는 슬롯'을 희생시켜 필요한
한자를 가리키게 합니다.

* 이번에 **번역하지 않는** 챕터의 사용량만 집계해 우선순위를 정하므로,
  번역하는 챕터가 많아질수록 더 안전해집니다.
* **상한: 약 1,930자.** 게임 전체 스크립트의 고유 음절이 이 이내면
  이 방식으로 충분합니다 (대부분의 경우 해당).
* 부작용: 희생된 슬롯을 쓰던 '아직 번역 안 한' 챕터에서 희귀 글자
  일부가 다르게 보일 수 있음 (전체 번역 시 해소).

### 4-2. 사전 확장 방식 (`--expand-dict`)

희생슬롯 1,930자로 모자랄 때 사용합니다.
`seg1` 의 코드 범위를 물리적으로 늘려 (압축 세그먼트 `seg2` 의 테이블을
빈 공간으로 이전) **한자 2,350자 전부**를 인코딩할 수 있게 합니다.

* 사전 헤더의 `seg1` 경계값과 `seg2` 테이블 위치만 수정 → 안정적.
* 검증 결과 압축 코드 12,557개 중 **7개(0.06%)** 만 영향
  (이동된 코드를 참조하던 극소수 압축 엔트리).
* **권장 상황: 전체 또는 거의 전체 챕터를 번역할 때.**
  미번역 챕터의 시나리오 텍스트에 약간의 글자 변화가 생길 수 있으나,
  모든 챕터를 번역하면 시나리오는 이 코드를 쓰지 않으므로 무관합니다.

`patch_all.py` 는 기본 방식에서 용량이 부족하면 `--expand-dict` 를
쓰라고 안내합니다.

## 5. 출력 읽는 법

* **희생슬롯 우선순위 계산** — 번역 안 하는 챕터 기준 사용량 집계.
* **사전 코드 배정** — 배정한 문자 수, 희생슬롯/확장코드 수.
  `★ 코드 미배정 문자` 가 나오면 용량 초과 → `--expand-dict` 사용.
* **챕터별 빌드 & 배치** — 각 챕터의 새 주소·크기·한국어 턴 수.
  `⚠ 미번역 N` 은 `번역 없음` 으로 표시된 턴 수,
  `⚠ 매칭점검 N` 은 한국어가 일본어보다 2줄 이상 길어진 턴 수
  (번역 매칭 엑셀의 턴 경계를 점검하면 좋음).
* **완료** — ROM 확장 여부, 최종 크기.
* **검증** — 패치 ROM 을 다시 읽어 모든 번역 챕터가 정상 파싱되는지
  확인하고, 첫 챕터 대사를 한글로 복원해 보여 줌.

## 7. 주의 사항

1. **항상 원본(클린) 한글패치 ROM 에서 패치하세요.**
   이미 패치한 ROM 에 다시 패치하면 안 됩니다.

2. **인덱스 단조성** — 블록을 빈 공간으로 옮기면 인덱스 오프셋이
   단조 증가하지 않게 됩니다. 게임이 인덱스를 '블록 시작 주소'로만
   쓰면 정상 (1화 실기 테스트로 확인됨). 이 도구의 분석기도 단조성에
   의존하지 않도록 되어 있습니다.

3. **줄 수 +2 이상 턴** — 번역 매칭 엑셀의 턴 경계가 ROM 턴과
   어긋난 부분입니다. 도구는 자동 확장+표시로 처리하지만, 매칭을
   다듬으면 더 깔끔합니다.

4. **에뮬레이터로 꼭 확인** — 패치 후 각 챕터를 진입해 대사가 한글로
   잘 나오는지, 대사창을 넘치지 않는지 확인하세요. 넘치면 `--reserve`
   값을 조정하거나 해당 턴 번역을 줄입니다.
